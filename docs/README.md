# docs/

每个开发阶段的设计 & 交付文档。

## 工作流

每个阶段开始前：

1. **先生成计划文档** `stage-XX-<slug>.md`（不开始编码），覆盖：背景/目标、技术决策、目录变更、关键模块设计、扩展点、实施步骤、验证方式
2. **输出文档给用户审阅**
3. **审阅通过后再按文档执行编码**
4. 阶段完成后，把实际产出回填同一份文档，作为最终交付物
